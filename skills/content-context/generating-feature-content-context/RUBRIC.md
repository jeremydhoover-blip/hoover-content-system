# Rubric

## Pass criteria
Pass if all are true:

- [ ] Feature purpose is clearly defined
- [ ] User goals are documented with priority
- [ ] Feature boundaries are explicit (what's in/out)
- [ ] All states are mapped (empty, loading, success, error, edge cases)
- [ ] Each state documents user need in that moment
- [ ] UI components are inventoried with character limits
- [ ] Business rules affecting content are documented
- [ ] Terminology is defined with usage guidance
- [ ] Terms to avoid are listed with alternatives
- [ ] Context is structured for AI consumption (consistent format)

## Fail criteria
Fail if any are true:

- [ ] Missing state coverage (especially error states)
- [ ] User goals not documented
- [ ] No terminology defined
- [ ] Business rules/constraints ignored
- [ ] Format is inconsistent or unstructured
- [ ] No character limits for UI components
- [ ] Missing "terms to avoid" guidance
- [ ] No version/update tracking
- [ ] Feature boundaries unclear (scope creep)
