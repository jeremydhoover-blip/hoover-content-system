# Journey Stage Taxonomy

## Standard journey stages

Use these as baseline stages. Adapt to product context but maintain the underlying intent of each stage.

### Acquisition journey stages

| Stage | User intent | Content focus | Success metric |
|-------|-------------|---------------|----------------|
| **Awareness** | Understanding what this product is | Value proposition, problem statement | Time on page, scroll depth |
| **Interest** | Evaluating if it might solve their problem | Features, benefits, social proof | CTA clicks, email signup |
| **Consideration** | Comparing against alternatives | Differentiation, pricing, case studies | Pricing page views, demo requests |
| **Decision** | Choosing to try or buy | Risk reduction, clear CTA | Trial signups, purchases |

### Activation journey stages

| Stage | User intent | Content focus | Success metric |
|-------|-------------|---------------|----------------|
| **Onboarding** | Learning how to get started | Setup guidance, quick wins | Onboarding completion rate |
| **First value** | Accomplishing first meaningful task | Task guidance, empty states | First action completion |
| **Habit formation** | Building the product into routine | Feature discovery, reminders | Return visits, feature adoption |

### Engagement journey stages

| Stage | User intent | Content focus | Success metric |
|-------|-------------|---------------|----------------|
| **Active use** | Accomplishing regular tasks | In-context help, efficiency tips | Session frequency, task completion |
| **Expansion** | Getting more value from the product | Cross-feature education, upgrade prompts | Feature breadth, upsells |
| **Mastery** | Becoming expert user | Advanced features, shortcuts | Advanced feature adoption |

### Retention journey stages

| Stage | User intent | Content focus | Success metric |
|-------|-------------|---------------|----------------|
| **Re-engagement** | Returning after absence | Win-back messaging, what's new | Return rate, reactivation |
| **Renewal** | Continuing subscription | Value recap, renewal prompts | Renewal rate |
| **Recovery** | Resolving issues blocking use | Error handling, support content | Issue resolution rate |

### Advocacy journey stages

| Stage | User intent | Content focus | Success metric |
|-------|-------------|---------------|----------------|
| **Satisfaction** | Feeling good about the product | Feedback requests, success celebration | NPS, satisfaction surveys |
| **Referral** | Recommending to others | Referral prompts, shareable content | Referral rate, shares |
| **Loyalty** | Long-term commitment | Loyalty rewards, exclusive access | LTV, tenure |

## Stage characteristics

Each stage should document:

| Attribute | Description | Example |
|-----------|-------------|---------|
| **User goal** | What the user is trying to accomplish | "Find a tool that helps manage team projects" |
| **Emotional state** | How the user feels at this stage | Curious, skeptical, frustrated, confident |
| **Information needs** | What questions they need answered | "How much does it cost?", "Can I import my data?" |
| **Barriers** | What might prevent progression | Unclear pricing, complex setup, missing feature |
| **Entry trigger** | What brings users to this stage | Marketing ad click, colleague referral, search query |
| **Exit criteria** | What marks completion of this stage | Account created, first task completed, payment submitted |

## Emotional state vocabulary

Use these terms consistently when documenting emotional state:

### Positive states
- **Confident:** Trusts the product/process
- **Curious:** Interested in learning more
- **Motivated:** Ready to take action
- **Relieved:** Problem solved or concern addressed
- **Satisfied:** Expectations met or exceeded

### Neutral states
- **Evaluative:** Weighing options objectively
- **Focused:** Concentrated on task
- **Patient:** Willing to wait
- **Uncertain:** Unsure but not negative

### Negative states
- **Frustrated:** Blocked or confused
- **Anxious:** Worried about outcome
- **Skeptical:** Doubting claims or value
- **Impatient:** Wants faster resolution
- **Overwhelmed:** Too much information or complexity

## Journey type classification

| Journey type | Scope | Typical stages | Example |
|--------------|-------|----------------|---------|
| **Full lifecycle** | Awareness through advocacy | 6-10 stages | New user acquisition to loyal advocate |
| **Acquisition** | Awareness through decision | 3-5 stages | Marketing funnel |
| **Activation** | First login through habit formation | 3-4 stages | Onboarding flow |
| **Task journey** | Single task start to completion | 2-4 stages | File upload, form submission |
| **Recovery** | Error/issue through resolution | 2-4 stages | Password reset, payment failure |
| **Expansion** | Current state through upgrade | 3-4 stages | Free to paid conversion |
