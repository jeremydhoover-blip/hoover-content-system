# Rubric

## Pass criteria
Pass if all are true:

- [ ] Journey scope is explicitly defined (full lifecycle or specific phase)
- [ ] Target user segment is identified
- [ ] All relevant journey stages are documented
- [ ] Each stage has a defined user goal
- [ ] Each stage has an identified emotional state
- [ ] Each stage has at least one content touchpoint
- [ ] Touchpoints specify channel (in-app, email, push, web, etc.)
- [ ] Touchpoints specify content type (headline, body, CTA, etc.)
- [ ] Touchpoints specify purpose (inform, persuade, instruct, reassure)
- [ ] Content gaps are explicitly documented with impact assessment
- [ ] Cross-channel touchpoints are included where applicable
- [ ] Stage entry triggers and exit criteria are documented
- [ ] Content dependencies between stages are identified

## Fail criteria
Fail if any are true:

- Journey scope is undefined or ambiguous
- User segment is not specified
- Any stage lacks documented touchpoints
- User goals missing for any stage
- Emotional state not considered
- Only in-app content mapped when email/push/web touchpoints exist
- Content gaps section missing or states "none" without justification
- Journey treated as strictly linear when loops or branches exist
- No distinction between user goals and business goals
- Touchpoints listed without channel or content type
- Recovery or error journeys completely ignored
- Cross-stage dependencies not documented for multi-stage features
