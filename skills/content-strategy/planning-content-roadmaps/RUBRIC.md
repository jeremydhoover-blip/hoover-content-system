# Rubric

## Pass criteria
Pass if all are true:

- [ ] Planning period is clearly defined (quarter or semester recommended)
- [ ] Business objectives are stated with explicit content connection
- [ ] User needs are documented with gap analysis
- [ ] Each initiative has defined scope, owner, target date
- [ ] Dependencies are identified with owners and dates
- [ ] Success metrics are specific and measurable
- [ ] Resource allocation is documented
- [ ] Timeline includes clear milestones
- [ ] Parking lot exists for deferred items with revisit triggers
- [ ] Review/update cadence is defined

## Fail criteria
Fail if any are true:

- [ ] No connection between content initiatives and business objectives
- [ ] Missing owners or dates for initiatives
- [ ] No success metrics defined
- [ ] Dependencies not identified
- [ ] Resource constraints ignored (over-allocation)
- [ ] No prioritization rationale
- [ ] Planning horizon too long (>6 months) without checkpoints
- [ ] No mechanism for handling unplanned requests
- [ ] Initiatives lack clear scope definition
