# Rubric

## Pass if all are true
- [ ] Headline clearly states what access is needed
- [ ] Headline is ≤60 characters
- [ ] Explanation describes why permission is needed
- [ ] Explanation shows user benefit (what they get by allowing)
- [ ] Explanation is ≤150 characters
- [ ] Allow label is ≤20 characters
- [ ] Deny label is ≤20 characters
- [ ] Deny option is always available (no forced permissions)
- [ ] Access denied state provides path to re-enable
- [ ] Permission requested is actually needed for the feature

## Fail if any are true
- [ ] No explanation of why permission is needed
- [ ] Explanation focuses only on app needs, not user benefit
- [ ] Forced permission (no deny option)
- [ ] Generic headline ("Permission needed")
- [ ] Character limits exceeded
- [ ] Denied state provides no recovery path
- [ ] Permission requested before user understands why
- [ ] Requesting permissions unrelated to current action
