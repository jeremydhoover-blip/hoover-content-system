# Banned Terms Reference

Terms prohibited in feature naming, setting labels, and UI copy. Organized by reason for ban.

---

## Category: Technical jargon

Terms that expose implementation details to users.

| Banned term | Reason | Alternatives |
|-------------|--------|--------------|
| Cache | Internal mechanism | Stored data, Saved content |
| Buffer | Technical concept | Temporary storage |
| Token | Auth implementation | Session, Access |
| Webhook | Developer term | Automated notification, Integration |
| API | Developer term | Connection, Integration |
| SDK | Developer term | Tools, Integration kit |
| Regex | Technical concept | Pattern, Pattern matching |
| Query | Database term | Search, Filter |
| Payload | Network term | Data, Content |
| Callback | Programming concept | Response, Notification |
| Instance | Technical term | Copy, Version |
| Thread | Programming/technical | Conversation, Chain |
| Parse | Technical operation | Read, Process |
| Render | Technical operation | Display, Show |
| Compile | Technical operation | Build, Create |
| Deprecated | Developer term | Legacy, Older version, No longer supported |

| Implement | Technical/vague | Add, Enable, Set up |

## Usage notes

### Context exceptions
Some banned terms acceptable in specific contexts:
- "API" acceptable in developer documentation
- "Webhook" acceptable in integration settings for technical users
- "Query" acceptable in database/analytics products

### Replacement process
1. Identify banned term in copy
2. Determine why term was used (what concept?)
3. Select alternative that preserves meaning
4. Verify alternative fits character limits
5. Check consistency with existing vocabulary

### New term proposals
To add terms to product vocabulary:
1. Define the term clearly
2. Specify where it will appear
3. Confirm no existing term covers concept
4. Verify it passes all ban categories
5. Document in product glossary
