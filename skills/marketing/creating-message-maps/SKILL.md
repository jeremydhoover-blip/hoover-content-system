# Creating message maps

## Quick start
Collect or infer:
- Product or feature name
- Target audience(s) and segments
- Core problem being solved
- Key differentiators (vs. alternatives)
- Proof points and evidence
- Business goals for the messaging

Then produce output using [TEMPLATES.md](TEMPLATES.md). Validate with [RUBRIC.md](RUBRIC.md).

## Workflow
1. Define the primary audience and their key pain point.
2. Draft the core message (single sentence value proposition).
3. Identify 3-4 supporting pillars (key themes or benefit categories).
4. For each pillar, add supporting messages and proof points.
5. Map messages to audience segments if multiple audiences.
6. Validate hierarchy: can someone understand value in 10 seconds?
7. Run the rubric check. Revise until it passes.

## Degrees of freedom
- **Low**: Core message and pillar count (3-4) are constrained.
- **Medium**: Supporting message phrasing, proof point selection.
