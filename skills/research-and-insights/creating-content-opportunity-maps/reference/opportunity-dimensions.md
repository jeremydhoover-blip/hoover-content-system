# Opportunity Map Dimensions

## Common dimension pairs

Choose dimension pairs based on what decision needs to be made:

| Decision context | X-axis | Y-axis |
|------------------|--------|--------|
| Resource allocation | Effort | Impact |
| Strategic planning | Strategic fit | Urgency |
| Gap analysis | Current coverage | User need intensity |
| Risk assessment | Confidence | Potential value |
| Quick wins | Time to value | Value magnitude |

---

## Impact dimensions

### User impact
| Score | Criteria |
|-------|----------|
| 5 | Solves critical blocker for >50% of users |
| 4 | Significantly improves experience for 25-50% |
| 3 | Measurable improvement for 10-25% |
| 2 | Minor improvement, affects <10% |
| 1 | Nice to have, minimal user benefit |

### Business impact
| Score | Criteria |
|-------|----------|
| 5 | Directly moves key business metric (revenue, retention, activation) |
| 4 | Strongly supports key metric |
| 3 | Contributes to business goals |
| 2 | Indirect or long-term business benefit |
| 1 | No clear business connection |

### Support impact
| Score | Criteria |
|-------|----------|
| 5 | Top 5 ticket driver, >500 tickets/month |
| 4 | Top 10 ticket driver, 200-500 tickets/month |
| 3 | Top 20 ticket driver, 50-200 tickets/month |
| 2 | Occasional support topic, <50 tickets/month |
| 1 | Rarely causes support contacts |

## Effort dimensions

### Content effort
| Score | Criteria |
|-------|----------|
| 1 | <1 day, minor edit or update |
| 2 | 1-3 days, single asset creation |
| 3 | 1-2 weeks, multiple assets or significant research |
| 4 | 2-4 weeks, cross-functional coordination needed |
| 5 | >1 month, major initiative |

### Cross-functional effort
| Score | Criteria |
|-------|----------|
| 1 | Content team can do independently |
| 2 | Needs SME input but no other team work |
| 3 | Needs design review or light engineering |
| 4 | Requires dedicated design or engineering |
| 5 | Requires significant engineering or new tooling |

### Maintenance effort
| Score | Criteria |
|-------|----------|
| 1 | Evergreen, rarely needs updates |
| 2 | Annual refresh sufficient |
| 3 | Quarterly updates needed |
| 4 | Monthly updates required |
| 5 | Continuous maintenance, tied to product changes |

## Strategic dimensions

### Strategic alignment
| Score | Criteria |
|-------|----------|
| 5 | Directly tied to top company priority |
| 4 | Supports primary team/department goal |
| 3 | Aligns with stated strategy |
| 2 | Indirectly related to strategy |
| 1 | No clear strategic connection |

### Urgency
| Score | Criteria |
|-------|----------|
| 5 | Blocking launch, deal, or major initiative |
| 4 | Needed this quarter |
| 3 | Needed this half |
| 2 | Needed this year |
| 1 | No time pressure |

### Competitive necessity
| Score | Criteria |
|-------|----------|
| 5 | Competitors all have it, we don't—table stakes |
| 4 | Most competitors have it |
| 3 | Some competitors have it |
| 2 | Differentiator—competitors lack |
| 1 | Not a competitive factor |

---

## Confidence dimensions

### Evidence strength
| Score | Criteria |
|-------|----------|
| 5 | Quantitative data, large sample, clear signal |
| 4 | Quantitative data with caveats OR strong qualitative |
| 3 | Moderate qualitative evidence |
| 2 | Limited evidence, reasonable inference |
| 1 | Assumption or hypothesis, needs validation |

### Execution confidence
| Score | Criteria |
|-------|----------|
| 5 | We've done this before, clear path |
| 4 | Similar to past work, minor unknowns |
| 3 | Some unknowns, but manageable |
| 2 | Significant unknowns, may need iteration |
| 1 | Novel, high uncertainty |

## Coverage dimensions (for gap analysis)

### Current coverage
| Level | Definition |
|-------|------------|
| None | No content exists for this need |
| Minimal | Something exists but doesn't address need |
| Partial | Content exists but incomplete or outdated |
| Adequate | Content meets basic need |
| Full | Comprehensive, high-quality coverage |

### User need intensity
| Level | Criteria |
|-------|----------|
| Critical | Users cannot succeed without this |
| High | Significant friction without this |
| Medium | Helpful but workarounds exist |
| Low | Nice to have |
| None | Users don't need this |

## Quadrant naming conventions

### Impact vs. Effort map
| Quadrant | Name | Recommendation |
|----------|------|----------------|
| High impact, Low effort | Quick Wins / Do Now | Prioritize immediately |
| High impact, High effort | Big Bets / Strategic Projects | Plan and resource carefully |
| Low impact, Low effort | Fill-ins / Opportunistic | Do if capacity allows |
| Low impact, High effort | Avoid / Time Sinks | Deprioritize |

### Strategic fit vs. Urgency map
| Quadrant | Name | Recommendation |
|----------|------|----------------|
| High strategic, High urgency | Critical Path | Prioritize immediately |
| High strategic, Low urgency | Strategic Backlog | Plan for future |
| Low strategic, High urgency | Tactical Needs | Minimize investment |
| Low strategic, Low urgency | Parking Lot | Deprioritize |

### Coverage vs. Need map
| Quadrant | Name | Recommendation |
|----------|------|----------------|
| High need, No coverage | Critical Gaps | Create immediately |
| High need, Full coverage | Maintain | Protect and update |
| Low need, No coverage | Nice to Have | Create if capacity |
| Low need, Full coverage | Over-invested | Consider deprecation |

---

## When to use which dimensions

| Situation | Recommended dimensions |
|-----------|------------------------|
| Limited resources, need quick wins | Impact × Effort |
| Strategic planning | Strategic alignment × Business impact |
| Content audit | Coverage × User need |
| Launch planning | Launch necessity × Time to create |
| Support cost reduction | Support impact × Creation effort |
| Confidence is low | Evidence strength × Potential impact |
