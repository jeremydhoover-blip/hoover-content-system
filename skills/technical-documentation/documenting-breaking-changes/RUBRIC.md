# Rubric

Pass if all are true:

- [ ] Change is described with specific technical details
- [ ] Before and after behavior clearly contrasted
- [ ] Affected users can self-identify with provided criteria
- [ ] Step-by-step migration instructions provided
- [ ] Each migration step has verification checkpoint
- [ ] Timeline includes specific dates
- [ ] Deadline for required action is clear
- [ ] Code examples show both old and new approach
- [ ] Consequences of not migrating are stated
- [ ] Support/help resources linked

Fail if any are true:

- Change described without explaining impact
- Missing migration steps
- Users cannot determine if they are affected
- Timeline uses relative terms ("soon", "later")
- No code examples for code-related changes
- Migration steps lack verification
- No clear deadline for action
- Assumes knowledge not stated in prerequisites
- Missing support resources
