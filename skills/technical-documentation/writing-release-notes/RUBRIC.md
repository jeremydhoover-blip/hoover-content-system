# Rubric

Pass if all are true:

- [ ] Version number and date clearly stated
- [ ] Breaking changes appear first when present
- [ ] Each breaking change includes migration instructions
- [ ] Breaking changes specify affected users/components
- [ ] Features describe user benefit, not just what was built
- [ ] Bug fixes describe the problem that was fixed
- [ ] Deprecations include replacement and removal timeline
- [ ] Security issues include severity rating
- [ ] Changes are categorized consistently
- [ ] Links provided for detailed documentation on complex changes

Fail if any are true:

- Breaking changes buried after features
- Breaking changes lack migration instructions
- Changes described only in technical/implementation terms
- Bug fixes say "fixed bug" without describing the problem
- Deprecations missing removal date
- Internal ticket numbers used as primary description
- Security issues missing severity
- Version or date missing
- Inconsistent categorization across similar items
