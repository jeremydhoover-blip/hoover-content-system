# Rubric

Pass if all are true:

- [ ] Title clearly identifies the component being documented
- [ ] Description is one sentence stating what the component does
- [ ] All parameters/options/properties are documented
- [ ] Each parameter includes type information
- [ ] Each optional parameter shows default value
- [ ] Constraints (ranges, valid values, max lengths) are documented
- [ ] At least one minimal usage example is provided
- [ ] Example demonstrates correct usage without solving a problem
- [ ] Related items are cross-referenced
- [ ] Formatting is consistent across all similar items

Fail if any are true:

- Parameters are missing type information
- Optional parameters lack default values
- Tutorial or task-oriented content is mixed in
- Examples are overly complex or solve specific problems
- Description exceeds two sentences
- Constraints are omitted for bounded values
- Different sections use inconsistent table column order
- Return values or outputs are undocumented
- Error conditions are omitted for APIs
