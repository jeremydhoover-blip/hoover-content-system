# Templates

## Default how-to guide structure

```md
# How to [accomplish specific task]

Brief description of what this guide helps you do and when to use it.

## Prerequisites

Before you begin, ensure you have:

- [ ] [Prerequisite 1]
- [ ] [Prerequisite 2]
- [ ] [Prerequisite 3]

## Steps

### Step 1: [Action verb] [object]

[One to two sentences explaining what this step does.]

```[language]
[code or command]
```

**Expected outcome:** [What the user should see or have after completing this step.]

### Step 2: [Action verb] [object]

[Instructions for step 2.]

> **Note:** [Important contextual information if needed.]

**Expected outcome:** [Result description.]

### Step 3: [Action verb] [object]

[Instructions for step 3.]

**Expected outcome:** [Result description.]

## Verify your setup

[How to confirm the task completed successfully.]

```[language]
[verification command or check]
```

You should see: [expected verification output].

## Next steps

- [Related guide 1]
- [Related guide 2]

## Troubleshooting

### [Common problem 1]

**Symptom:** [What the user observes.]

**Solution:** [How to fix it.]

### [Common problem 2]

**Symptom:** [What the user observes.]

**Solution:** [How to fix it.]
```

## Short-form template (3 steps or fewer)

```md
# How to [accomplish specific task]

[One-sentence description.]

**Prerequisites:** [Inline prerequisites if minimal.]

1. [Step 1 with action and outcome.]
2. [Step 2 with action and outcome.]
3. [Step 3 with action and outcome.]

**Verify:** [Quick verification method.]
```

## Variation rules
- Use short-form only when prerequisites are minimal and steps are 3 or fewer.
- Code blocks must specify language.
- Expected outcome is required for each step in default template.
- Troubleshooting section is optional but recommended for guides involving system configuration.
